# YeBetWeg Repository Deep-Dive Analysis & Development Roadmap

## Executive Summary

YeBetWeg is a production-ready construction knowledge platform and marketplace for Ethiopia, built with React 19, TypeScript, Vite 7, Tailwind CSS v4, and Supabase. The platform serves bilingual content (English/Amharic) with 9 database tables, 60+ UI components, and implements Row Level Security (RLS) for data protection.

---

## 1. ARCHITECTURAL ANALYSIS

### 1.1 Technology Stack
| Layer | Technology | Version | Purpose |
|-------|-----------|---------|---------|
| Frontend | React | 19.0.0 | Component framework |
| | TypeScript | 5.9.3 | Type safety |
| | Vite | 7.3.1 | Build tool |
| | Tailwind CSS | 4.2.1 | Styling (OKLCH colors) |
| | shadcn/ui | 60+ components | UI library |
| Backend | Supabase | PostgreSQL | Database & auth |
| | RLS | - | Row-level security |

### 1.2 Project Structure
```
src/
├── components/
│   ├── layout/          # Navbar, Footer, FloatingSocialBar
│   ├── sections/        # 10 page sections (Hero, Blog, Tips, etc.)
│   ├── icons/           # Custom SVG icons
│   └── ui/              # 60+ shadcn components
├── hooks/               # 7 data fetching hooks
├── lib/                 # i18n, supabase, utils, validation
├── context/             # AuthContext for authentication
└── App.tsx              # Main application
```

### 1.3 Database Schema (9 Tables)
| Table | Records | Purpose | RLS Policy |
|-------|---------|---------|------------|
| blogs | 8 | Construction articles | Public read |
| tips | 10 | Expert tips (6 free, 4 premium) | Public read |
| market_prices | 15 | Material pricing | Public read |
| listings | 12 | Marketplace items | Public read/insert |
| professionals | 6 | Verified pros | Public read/insert |
| ads | 3 | Advertisements | Public read (active) |
| subscribers | - | Newsletter signups | Public insert |
| inquiries | - | Contact requests | Public insert |
| premium_subscriptions | - | Membership tracking | Authenticated read |

## 2. CORE BUSINESS LOGIC

### 2.1 Bilingual Implementation
- Custom i18n Context (120+ translation keys)
- All content stored bilingually (title_am/title_en)
- Noto Sans Ethiopic font for Amharic rendering
- Language preference persisted in localStorage

### 2.2 Premium Gating Mechanism
- **Tips**: 6 free, 4 premium (glassmorphism blur overlay)
- **Market Prices**: 5 free rows, rest blurred
- **Payment Integration**: Chapa/TeleBirr buttons (UI-ready, no backend)

### 2.3 Data Flow Pattern
```
Component → useInView hook → Supabase query → Loading states → Render
```

### 2.4 Security Features
- RLS policies on all tables
- Input validation/sanitization (prevent SQL injection, XSS)
- Form validation with Zod-like utilities
- Protected contact info (no direct display)

## 4. TECHNICAL DEBT & ISSUES

### 4.1 Code Issues Identified
1. **Navbar.tsx:24** - Uses undefined `useAuth()` instead of `useAuthContext()`
2. **useAuth.ts** - Duplicate of AuthContext logic (both exist independently)
3. **MarketplaceSection.tsx** - Inquiry submission is mock (no data persisted)
4. **ProfessionalsSection.tsx** - Hire inquiry doesn't persist to database
5. **Missing components**: Footer.tsx, FloatingSocialBar.tsx, mode-toggle.tsx, theme-provider.tsx

### 4.2 Security Concerns
- No CSRF protection on form submissions
- No rate limiting on public insert endpoints
- Contact phone/email not protected (RLS allows anyone to read)
- No input sanitization on all form fields
- Missing authentication guard on protected routes
